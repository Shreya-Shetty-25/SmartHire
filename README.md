# SmartHire

SmartHire is a smart HR recruitment system with a React+Vite frontend and a FastAPI backend using PostgreSQL (local dev).

## Structure

- frontend/ - React + Vite (JavaScript)
- backend/ - FastAPI (Python) + PostgreSQL

## Local development

- Start the backend from `backend/` on the default FastAPI port `8000`:
  `..\ .venv\Scripts\python.exe -m uvicorn app.main:app --reload`
- Start the frontend from `frontend/` on port `5173`:
  `npm run dev`
- In local dev, the frontend uses Vite proxy:
  - `/api` -> `http://127.0.0.1:8000`
  - `/assessment-api` -> `http://127.0.0.1:8000`
- You usually do not need to set `VITE_API_BASE_URL` locally. The default is `/api`.

## Backend configuration (local Postgres)

- In backend/, copy .env.example to .env and set DATABASE_URL.
- Start Postgres locally (ensure the user/password/db in DATABASE_URL exist).

## Candidates + resume parsing

- The backend exposes authenticated endpoints under /api/candidates.
- Resume upload uses an LLM provider (Azure OpenAI / Gemini / Groq) selected via boolean env flags.
- Configure exactly one of USE_AZURE_OPENAI / USE_GEMINI / USE_GROQ in backend/.env (see backend/.env.example).

## Voice calling (Twilio)

This repo now includes a demo voice-calling endpoint that places an outbound call to a candidate and runs a short 5-step conversation:

- HR -> user -> HR -> user -> HR (then hang up)

Each HR turn is generated by the configured LLM provider (Groq/Gemini/Azure OpenAI) and spoken using ElevenLabs.

Backend env vars (set in backend/.env):

- TWILIO_ACCOUNT_SID
- TWILIO_AUTH_TOKEN
- TWILIO_FROM_NUMBER
- PUBLIC_BASE_URL (must be a publicly reachable URL, e.g. an ngrok URL, so Twilio can fetch the TwiML webhook)

ElevenLabs env vars (set in backend/.env):

- ELEVENLABS_API_KEY
- ELEVENLABS_VOICE_ID
- ELEVENLABS_MODEL_ID (optional; default is eleven_multilingual_v2)

Endpoints:

- POST /api/calls/voice/demo with JSON: {"phone_number": "+1...", "position": "...", "candidate_name": "..."}
- GET /api/calls/voice/twiml?name=...&position=... (Twilio webhook; returns TwiML)

Notes:

- The voice agent uses the same LLM provider selection flags as resume parsing (exactly one of USE_AZURE_OPENAI / USE_GEMINI / USE_GROQ).
- If no LLM provider is enabled, it falls back to deterministic scripted lines so the call can still proceed.
